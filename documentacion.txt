P3 – Práctica Bloque III

Se trata de implementar una aplicación tipo SPA mediante el uso intensivo de Ajax.

Requisitos técnicos:

Carga dinámica de tres formularios y scripts de dichos formularios
Uso de localStorage para cacheo de datos de la BB.DD. : provincias, localidades, etc..
[ Uso de diálogos y datepicker de Jquery UI. ] Pendiente
Presencia de al menos una llamada ajax de cada tipo de las siguientes:
.load ---->La carga de todos los dialogos index.js , la carga de los datos a los combobox (Reparaciones)
$.get
$.post---->Carpeta clientes las llamadas ajax son $.Post con los datos serializados ( .serialize)
$.ajax
$.getScript-----> la carga de los scrips de los js de cada apartado, index js
Ajax sin jquery
Recuperación/envío de datos con el servidor en las llamadas Ajax en los siguientes formatos:
HTML---> la carga de datos de los combobox ( en proceso), en reparaciones
Script javascript
JSON
XML
Envío de datos serializados (tipo p1=v1&p2=v2…)
Subida del proyecto finalizado a GitHub

Material para la entrega:
Modelo E/R documentado
Documento word con información de lo que almacena cada tabla de la BBDD
Documento Word o pdf con el diagrama E/R

Copia de la base de datos en formato .sql, conteniendo datos de prueba

Código fuente de la aplicación 

Documento word con la descripción de la práctica, en concreto se necesita conocer:

6 funciones de actualización de datos (alta de cliente, modificación de contrato, realizar reserva, …)
6 funciones para generación de listados, al menos tres con filtrado de datos mediante formulario.
Localización de los usos de Ajax y formatos de envio/recepción de datos.
Por ejemplo:	
$.ajax -> altaCasa.js línea 1500
XML -> listadoClientes.js línea 234
